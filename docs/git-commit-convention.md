# Git 提交规范

## 提交格式

```
<type>[optional scope]: <description>

[optional body]
```

## 提交类型 (Type)

### 主要类型

- **feat**: 新功能（feature）
- **fix**: 修复 bug
- **docs**: 文档变更
- **style**: 代码格式（不影响代码运行的变动，如空格、格式化、缺少分号等）
- **refactor**: 重构（既不是新增功能，也不是修复 bug 的代码变动）
- **perf**: 性能优化
- **test**: 增加测试或修改现有测试
- **build**: 影响构建系统或外部依赖的变更（如 Maven、Gradle、npm 等）
- **ci**: CI 配置文件和脚本的变更（如 GitHub Actions、Jenkins 等）
- **chore**: 其他变更（不修改 src 或 test 文件的变动，如更新依赖、配置文件等）
- **revert**: 回滚之前的提交

### 类型说明

| 类型     | 说明     | 示例                                       |
| -------- | -------- | ------------------------------------------ |
| feat     | 新增功能 | `feat: 添加用户登录功能`                 |
| fix      | 修复 bug | `fix: 修复登录时密码验证失败的问题`      |
| docs     | 文档变更 | `docs: 更新 API 文档`                    |
| style    | 代码格式 | `style: 格式化代码，统一缩进`            |
| refactor | 重构代码 | `refactor: 重构用户服务类，提取公共方法` |
| perf     | 性能优化 | `perf: 优化数据库查询性能`               |
| test     | 测试相关 | `test: 添加用户登录单元测试`             |
| build    | 构建系统 | `build: 更新 Maven 依赖版本`             |
| ci       | CI 配置  | `ci: 添加 GitHub Actions 工作流`         |
| chore    | 其他变更 | `chore: 更新 .gitignore 文件`            |
| revert   | 回滚提交 | `revert: 回滚 feat: 添加新功能的提交`    |

## 作用域 (Scope)

作用域是可选的，用于说明提交影响的范围。常见的作用域包括：

- **模块名**: 如 `user`, `auth`, `order` 等
- **组件名**: 如 `controller`, `service`, `dao`, `config` 等
- **文件名**: 如 `UserController`, `AuthService` 等

### 作用域示例

```
feat(user): 添加用户注册功能
fix(auth): 修复登录验证逻辑
docs(api): 更新用户接口文档
refactor(service): 重构订单服务类
```

## 描述 (Description)

描述是必填的，应该：

- 使用中文或英文（建议统一使用中文）
- 简短描述（不超过 50 个字符）
- 使用祈使句，以动词开头（如：添加、修复、更新、删除等）
- 首字母小写
- 结尾不加句号

### 描述示例

```
feat: 添加用户登录功能
fix: 修复密码验证失败的问题
docs: 更新 README 文档
refactor: 重构用户服务类
```

## 正文 (Body)

正文是可选的，用于详细说明：

- 提交的动机
- 与之前行为的对比
- 实现细节

正文应该：

- 在描述后空一行
- 每行不超过 72 个字符
- 使用中文或英文

### 正文示例

```
feat(user): 添加用户注册功能

实现了用户注册的核心功能，包括：
- 用户名和邮箱唯一性验证
- 密码强度校验
- 发送注册确认邮件
```

## 完整示例

### 示例 1: 简单提交

```
feat: 添加用户登录功能
```

### 示例 2: 带作用域的提交

```
fix(auth): 修复登录时 token 过期的问题
```

### 示例 3: 带正文的提交

```
feat(user): 添加用户注册功能

实现了用户注册的核心功能，包括：
- 用户名和邮箱唯一性验证
- 密码强度校验
- 发送注册确认邮件
```

### 示例 4: 详细说明的提交

```
fix(api): 修复用户查询接口返回空数据的问题

修复了当用户不存在时返回 null 导致的空指针异常
现在会返回空对象而不是 null，避免客户端出现空指针异常
```

### QA: 一次提交包含多个功能怎么办？

A: 应该拆分成多个提交，每个提交只包含一个功能或修复。

### Q: 如何回滚提交？

A: 使用 `revert` 类型：

```
revert: 回滚 feat: 添加用户登录功能的提交
```

### Q: 提交信息写错了怎么办？

A: 如果还没有推送到远程仓库，可以使用 `git commit --amend` 修改最后一次提交。

### Q: 如何查看提交历史？

A: 使用 `git log` 查看提交历史，推荐使用：

```bash
git log --oneline --graph --decorate
```
