# 前端 JSONB 数组字段处理指南

## 概述

在项目经历接口中，`techStack`（技术栈）和 `highlights`（项目亮点）字段需要以 **JSON 字符串格式** 发送到后端。

这些字段在数据库中存储为 PostgreSQL 的 JSONB 数组类型，但在 API 请求中需要以 JSON 字符串的形式传递。

## 字段说明

### techStack（技术栈）

- **类型**：字符串（JSON 数组格式）
- **格式要求**：必须是有效的 JSON 数组字符串
- **示例值**：`"[\"React\", \"TypeScript\", \"Node.js\"]"`

### highlights（项目亮点）

- **类型**：字符串（JSON 数组格式）
- **格式要求**：必须是有效的 JSON 数组字符串
- **示例值**：`"[\"实现了性能优化\", \"解决了分布式事务问题\"]"`

## 数据格式要求

### ✅ 正确格式

这些字段必须是 **JSON 数组的字符串表示**，即：

1. **外层是字符串**：整个值需要用双引号包裹
2. **内层是 JSON 数组**：数组中的字符串元素需要用转义的双引号 `\"` 包裹
3. **数组元素之间用逗号分隔**

### ❌ 错误格式

以下格式都是**错误的**，会导致后端解析失败：

1. **直接使用数组**（不是字符串）- 不能直接传递 JavaScript 数组对象
2. **普通字符串**（不是 JSON 数组格式）- 不能是逗号分隔的普通字符串
3. **缺少转义引号** - 数组中的每个元素都必须是字符串，需要用引号包裹
4. **空数组格式错误** - 空数组也必须是字符串格式 `"[]"`，不能是空数组对象

## 转换规则

### 从数组转换为 JSON 字符串

前端通常使用数组（如 `["React", "TypeScript"]`），需要转换为 JSON 字符串格式：

**转换步骤**：
1. 将数组序列化为 JSON 字符串
2. 确保字符串中的引号被正确转义
3. 整个结果作为字符串值传递

**转换示例**：
- 输入数组：`["React", "TypeScript", "Node.js"]`
- 输出字符串：`"[\"React\", \"TypeScript\", \"Node.js\"]"`

### 从 JSON 字符串转换为数组

后端返回的数据中，这些字段是 JSON 字符串格式，前端需要解析：

**解析步骤**：
1. 接收字符串值：`"[\"React\", \"TypeScript\"]"`
2. 使用 JSON 解析方法解析为数组
3. 得到数组：`["React", "TypeScript"]`

## 特殊情况处理

### 空数组

如果技术栈或亮点为空，可以：

1. **不传递字段**（推荐）- 在请求体中完全省略该字段
2. **传递空数组字符串** - 传递 `"[]"` 字符串
3. **传递 null** - 传递 `null` 值

### 单个元素

单个元素的数组也需要遵循数组格式，不能直接传递单个字符串值。

## 数据验证

在发送请求前，确保：

1. ✅ 字符串是有效的 JSON 格式
2. ✅ 数组中的每个元素都是字符串类型
3. ✅ 所有引号都已正确转义
4. ✅ 数组格式正确（以 `[` 开始，以 `]` 结束）

## 常见问题

### Q1: 为什么不能直接传数组？

**A**: 因为数据库字段类型是 JSONB，后端需要将数组转换为 JSON 字符串才能存储。如果直接传数组，后端无法正确处理。

### Q2: 如何验证格式是否正确？

**A**: 可以使用 JSON 解析方法测试，如果能成功解析为数组，说明格式正确。

### Q3: 如果数组包含特殊字符怎么办？

**A**: JSON 序列化工具会自动处理特殊字符的转义，无需手动处理。例如包含引号的字符串会被正确转义。

### Q4: 更新时如何处理？

**A**: 更新时同样需要传递 JSON 字符串格式。如果只更新部分字段，未更新的字段可以不传递。

## 接口字段对照表

|------------|----------|---------|---------|---------|
| 技术栈 | `techStack` | String | JSON 数组字符串 | 否 |
| 项目亮点 | `highlights` | String | JSON 数组字符串 | 否 |

## 注意事项

1. **不要手动拼接字符串**：使用 JSON 序列化工具来生成 JSON 字符串，避免格式错误

2. **保持一致性**：确保所有数组字段都使用相同的格式

3. **空值处理**：如果字段为空，可以不传递该字段，或传递 `null`，或传递 `"[]"`

4. **字符编码**：确保使用 UTF-8 编码，中文字符会被正确编码

5. **更新操作**：更新时，如果不想修改这些字段，可以不传递；如果想清空，传递 `"[]"` 或 `null`

6. **数组元素类型**：数组中的元素必须是字符串类型，不能是数字、布尔值或其他类型

7. **引号转义**：数组中的字符串元素必须使用转义的双引号 `\"`，不能使用单引号
